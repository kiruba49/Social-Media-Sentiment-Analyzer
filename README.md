Data PreparationCleaning: Removal of noise (punctuation, special characters) and standardization (lowercasing).Text Normalization: Applying Lemmatization to reduce words to their base form (e.g., 'running' $\rightarrow$ 'run') for consistent feature representation.2. Feature EngineeringVectorization: Conversion of cleaned text into numerical feature vectors using the TF-IDF (Term Frequency-Inverse Document Frequency) technique. This penalizes common words (like 'the') and weights important, differentiating words (like 'amazing' or 'terrible') higher.3. Model Training and PersistenceA Multinomial Naive Bayes classifier is trained on the TF-IDF features.The trained model (sentiment_model.pkl) and the fitted vocabulary (tfidf_vectorizer.pkl) are saved using the pickle library, allowing the model to be loaded instantly for real-time predictions without retraining.4. Evaluation and VisualizationThe model's performance is assessed using an Accuracy Score and a detailed Classification Report.A Confusion Matrix is generated to show performance across the Positive, Negative, and Neutral classes.A Word Cloud is created to visually confirm the most frequent and influential terms driving the positive classification.üöÄ How to Run the Project LocallyFollow these steps to set up and run the analysis on your machine:Clone the repository:Bashgit clone https://github.com/kiruba49/Social-Media-Sentiment-Analyzer.git
